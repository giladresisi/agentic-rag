# Module 1: App Shell + Observability

## Complexity: ⚠️ Medium

**Rationale:** Foundation module with multiple interconnected systems (auth, database, frontend, backend, observability). Each component is well-defined, but integration points require careful sequencing. SSE streaming and LangSmith tracing add moderate complexity.

## Context

Building the foundation for the Agentic RAG Masterclass - a working chat application with managed RAG using OpenAI's Responses API. This module establishes:

- **Authentication** via Supabase Auth with Row-Level Security
- **Chat interface** with React + TypeScript, supporting threaded conversations
- **Backend API** using Python FastAPI with SSE streaming
- **Managed RAG** via OpenAI Responses API (handles threads + file_search)
- **Observability** via LangSmith for all LLM interactions

**Current state:** Greenfield project - only documentation exists (CLAUDE.md, PRD.md, PROGRESS.md, README.md)

**Goal:** By the end of this module, users can sign up, create chat threads, send messages with streaming responses, and see every LLM interaction traced in LangSmith.

## Database Schema

### Tables

**threads**
- `id` (UUID, PK)
- `user_id` (UUID, FK to auth.users)
- `openai_thread_id` (TEXT, unique) - OpenAI's managed thread ID
- `title` (TEXT)
- `created_at` (TIMESTAMPTZ)
- `updated_at` (TIMESTAMPTZ)

**messages**
- `id` (UUID, PK)
- `thread_id` (UUID, FK to threads)
- `user_id` (UUID, FK to auth.users)
- `openai_message_id` (TEXT, nullable) - OpenAI's message ID
- `role` (TEXT, CHECK: 'user' or 'assistant')
- `content` (TEXT)
- `created_at` (TIMESTAMPTZ)

### RLS Policies
All tables must have policies for SELECT, INSERT, UPDATE, DELETE that enforce:
```sql
auth.uid() = user_id
```

### Indexes
- `threads(user_id)` - fast user thread lookup
- `threads(created_at DESC)` - chronological ordering
- `messages(thread_id)` - fast message retrieval
- `messages(created_at)` - chronological ordering

## Key Implementation Details

### OpenAI Responses API Integration

**Create and wrap client:**
```python
from openai import OpenAI
from langsmith.wrappers import wrap_openai

client = OpenAI(api_key=settings.OPENAI_API_KEY)
client = wrap_openai(client)  # Enable LangSmith tracing
```

**Thread operations:**
```python
# Create thread
thread = client.beta.threads.create()

# Add message and stream
client.beta.threads.messages.create(
    thread_id=thread_id,
    role="user",
    content=content
)

# Stream response
with client.beta.threads.runs.stream(
    thread_id=thread_id,
    assistant_id=settings.OPENAI_ASSISTANT_ID,
) as stream:
    for event in stream:
        if event.event == 'thread.message.delta':
            yield event.data.delta.content[0].text.value
```

### SSE Streaming (Backend)

```python
from sse_starlette.sse import EventSourceResponse

async def event_generator():
    full_response = ""

    async for delta in openai_service.stream_message(thread_id, content):
        full_response += delta
        yield {
            "event": "message",
            "data": json.dumps({"type": "content_delta", "delta": delta})
        }

    await save_message(thread_id, user_id, "assistant", full_response)
    yield {"event": "message", "data": "[DONE]"}

return EventSourceResponse(event_generator())
```

### SSE Streaming (Frontend)

```typescript
const sendMessage = async (content: string) => {
  const response = await fetch(`${API_URL}/chat/threads/${threadId}/messages`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Accept': 'text/event-stream',
    },
    body: JSON.stringify({ content }),
  });

  const reader = response.body?.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const lines = chunk.split('\n');

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const data = line.slice(6);
        if (data === '[DONE]') return;

        const parsed = JSON.parse(data);
        if (parsed.type === 'content_delta') {
          setStreamingContent(prev => prev + parsed.delta);
        }
      }
    }
  }
};
```

## Common Pitfalls to Avoid

1. **venv on Windows:** Use `venv\Scripts\activate`, not `venv/bin/activate`
2. **CORS errors:** Ensure `CORS_ORIGINS` includes frontend URL
3. **RLS testing:** Create 2 test users to verify data isolation
4. **SSE buffering:** Use `async` generators, avoid blocking operations
5. **LangSmith tracing:** Verify `LANGSMITH_TRACING=true` and use `wrap_openai()`
6. **OpenAI thread_id:** Store immediately after creation to prevent drift
7. **Token refresh:** Implement Supabase `onAuthStateChange` listener
8. **Assistant creation:** Create via OpenAI dashboard, enable `file_search` tool
