# 5. Document Ingestion Pipeline with Multi-Format Support

**Complexity:** 🔴 **Complex** - Requires database extensions, storage setup, multi-format parsing, chunking, embedding, and realtime updates

## Context

Phase 1 migrated to Chat Completions API. Now we build the document ingestion pipeline to:
- Store and process user-uploaded documents
- Generate embeddings for semantic search
- Prepare data for RAG retrieval (Phase 3)

## What This Phase Delivers

- Ingestion UI with drag-and-drop file upload
- Multi-format support: PDF, DOCX, HTML, Markdown, TXT (via docling)
- Supabase Storage bucket for raw files
- Text chunking with configurable strategy
- Embedding generation (OpenAI text-embedding-3-small)
- pgvector storage with metadata
- Realtime status updates during ingestion
- Document management (list, delete, view chunks)

## Database Changes

### 1. Enable pgvector (`supabase/migrations/004_enable_pgvector.sql` - NEW)
```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

### 2. Documents and Chunks Tables (`supabase/migrations/005_documents_and_chunks.sql` - NEW)

**Documents table:**
- id, user_id, filename, content_type, file_size_bytes, storage_path
- status (processing/completed/failed), error_message, chunk_count
- created_at, updated_at

**Chunks table:**
- id, document_id, user_id, content, embedding (vector(1536)), chunk_index, metadata
- created_at

**Indexes:**
- Performance indexes on user_id, status, created_at
- ivfflat index on embedding for vector similarity search

**RLS Policies:**
- Users can only view/create/update/delete their own documents and chunks

### 3. Supabase Storage Bucket
- Bucket name: `documents`
- RLS: Users can only read/write their own files

## Backend Changes

### 1. Configuration (`backend/config.py`)
Add document processing settings:
```python
EMBEDDING_MODEL: str = "text-embedding-3-small"
CHUNK_SIZE: int = 1000  # characters
CHUNK_OVERLAP: int = 200  # characters
MAX_FILE_SIZE_MB: int = 10
SUPPORTED_FILE_TYPES: List[str] = [".txt", ".pdf", ".docx", ".html", ".md"]
```

### 2. Dependencies (`backend/requirements.txt`)
Add: `docling==0.4.0` for multi-format document parsing

### 3. Pydantic Models (`backend/models/document.py` - NEW)
Define Document, Chunk, UploadDocumentRequest, DocumentResponse models

### 4. Embedding Service (`backend/services/embedding_service.py` - NEW)
**Purpose:** Document parsing, chunking, and embedding generation

**Key methods:**
- `parse_document(file_path, file_type)` - Parse any supported format to text using docling
- `chunk_text(text, chunk_size, overlap)` - Split text into overlapping chunks
- `generate_embeddings(texts, model)` - Generate embeddings for batch of texts

### 5. Ingestion Router (`backend/routers/ingestion.py` - NEW)
**Endpoints:**
- `POST /ingestion/upload` - Upload and process document
- `GET /ingestion/documents` - List all documents for user
- `GET /ingestion/documents/{document_id}` - Get document details
- `GET /ingestion/documents/{document_id}/chunks` - Get chunks
- `DELETE /ingestion/documents/{document_id}` - Delete document and chunks

**Upload flow:**
1. Validate file size and type
2. Upload to Supabase Storage
3. Create document record with status='processing'
4. Start background processing (async task)
5. Parse document → Chunk text → Generate embeddings → Save to DB
6. Update document status to 'completed' or 'failed'

### 6. Main App Update (`backend/main.py`)
Include ingestion router: `app.include_router(ingestion.router, prefix="/ingestion")`

## Frontend Changes

### 2. Ingestion Hook (`frontend/src/hooks/useIngestion.ts` - NEW)
**Purpose:** Manage document uploads and Realtime status subscriptions

**Features:**
- Fetch documents on mount
- Subscribe to Realtime updates for document status changes
- `uploadDocument(file)` - Upload file to backend
- `deleteDocument(documentId)` - Delete document
- `refreshDocuments()` - Refresh document list

**Realtime subscription:**
```typescript
supabase
  .channel('document-updates')
  .on('postgres_changes', {
    event: '*',
    schema: 'public',
    table: 'documents'
  }, handleUpdate)
  .subscribe();
```
