### 3. Database Function (`supabase/migrations/006_retrieval_function.sql` - NEW)
**Purpose:** pgvector similarity search function

```sql
CREATE OR REPLACE FUNCTION match_chunks(
    query_embedding vector(1536),
    match_threshold float,
    match_count int,
    user_id_filter uuid
)
RETURNS TABLE (
    id uuid,
    document_id uuid,
    content text,
    chunk_index integer,
    metadata jsonb,
    similarity float
)
```

**Logic:**
- Uses cosine similarity: `1 - (embedding <=> query_embedding)`
- Filters by user_id (RLS enforcement)
- Filters by similarity threshold
- Orders by similarity (ascending distance)
- Limits results

## How RAG Works End-to-End

1. User uploads document (Phase 2)
2. Document is chunked and embedded
3. Chunks stored in pgvector
4. User asks question in chat
5. LLM decides to use retrieve_documents tool
6. Query embedding generated
7. pgvector similarity search finds relevant chunks
8. Retrieved chunks injected into conversation context
9. LLM generates answer using retrieved context
10. Sources displayed in UI
