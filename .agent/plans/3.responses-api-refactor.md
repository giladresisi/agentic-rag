# Refactor to Responses API and Remove All Assistants API Leftovers

**Complexity:** ⚠️ Medium

## Goal
Properly implement OpenAI's Responses API (not Chat Completions API) with vector store support, and remove all remaining references to the deprecated Assistants API from code and documentation.

## Changes Required

### Backend Code
1. **openai_service.py** - Switch from Chat Completions API to Responses API
   - Use `client.responses.stream()` instead of `client.chat.completions.create()`
   - Add vector store support via `file_search` tool
   - Update event handling for Responses API format

2. **config.py** - Already has OPENAI_VECTOR_STORE_ID (good!)

3. **models/message.py** - Remove `openai_message_id` field (Assistants API leftover)

### Database
4. **Verify migration 002** - Ensure `openai_thread_id` column is removed
5. **Create migration 003** - Remove `openai_message_id` column from messages table

### Documentation
6. **Clean up all .md files** - Remove Assistants API references from:
   - MANUAL_TESTING_GUIDE.md
   - PROGRESS.md
   - SETUP.md
   - .agent/plans/*.md (keep as historical record but update status)

### Frontend
7. **Clean up comments/references** in TypeScript files

## Validation
- Backend starts successfully
- Can create threads
- Can send messages with streaming
- Vector store queries work (if OPENAI_VECTOR_STORE_ID is set)
- All Playwright tests pass
